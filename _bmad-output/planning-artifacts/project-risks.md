# Риски и ограничения проекта

## Известные ограничения проекта

### 1. Однопоточная архитектура auth-gateway.py

auth-gateway.py использует `http.server.BaseHTTPRequestHandler` с потоковой обёрткой (`ThreadedHTTPServer`). Каждый запрос обрабатывается в отдельном daemon-потоке. Это работает для учебного контейнера (1 студент), но имеет ограничения:

- **GIL Python** — потоки не дают истинной параллельности для CPU-bound задач
- **Нет connection pooling** к бэкендам — новое TCP-соединение на каждый запрос
- **Нет graceful shutdown** для WebSocket-соединений (relay_sockets просто обрывается)
- **Нет rate limiting** — любой клиент может забить gateway запросами

**Влияние на фичу провайдеров:** нулевое. auth-gateway не взаимодействует с API-провайдерами.

### 2. Статический claude-settings.json

`claude-settings.json` копируется в контейнер при сборке (Dockerfile строка 117) и содержит захардкоженные значения провайдера в секции `env`. Приоритет этого файла для Claude Code CLI не до конца ясен — возможно, его `env` перекрывает `.claude/.env`.

**Влияние на фичу провайдеров:** КРИТИЧЕСКОЕ. Если settings.json имеет приоритет над `.claude/.env`, то переключение провайдера через скрипт не будет работать. Необходимо исследовать поведение Claude Code CLI.

### 3. Генерация switch-api-key.sh внутри entrypoint.sh

Скрипт `switch-api-key.sh` не существует как файл в репозитории — он генерируется entrypoint.sh при каждом старте контейнера (heredoc на строках 33-52). Это значит:

- Изменения в скрипте требуют редактирования entrypoint.sh
- Скрипт использует `sed -i` для замены в `.claude/.env` — хрупко при изменении формата
- Скрипт не обновляет `claude-settings.json`

**Влияние на фичу провайдеров:** нужен новый подход. switch-provider.sh должен менять не только ключ, но и BASE_URL + модели, и желательно быть отдельным файлом в репозитории.

### 4. Привязка .env.example к конкретному провайдеру

Переменные в `.env.example` используют префикс `GLM_` (GLM_API_KEY, GLM_OPUS_MODEL). Это:

- Сбивает с толку при использовании другого провайдера
- Маппинг `GLM_API_KEY` на `ANTHROPIC_AUTH_TOKEN` происходит в docker-compose.yml (строка 14)
- Имена моделей (`GLM-5`, `GLM-4.5-Air`) специфичны для Z.AI

### 5. Отсутствие тестов

В проекте нет ни одного теста. Нет:

- Unit-тестов для auth-gateway.py
- Интеграционных тестов для entrypoint.sh
- Smoke-тестов для контейнера
- Проверки, что `.claude/.env` генерируется корректно

**Влияние:** любые изменения могут сломать проект незаметно. Особенно опасно для entrypoint.sh — ошибка приводит к нестартующему контейнеру.

### 6. Volume mount перезаписывает course/

При `docker compose up` volume `student-data` монтируется в `/home/coder/course/`. При первом запуске entrypoint.sh копирует из `.course-image/`. При повторном — студенческие данные сохраняются. Но:

- Обновление курсовых материалов требует `docker compose down -v` (удаляет данные студента)
- Нет механизма обновления материалов без потери данных

### 7. Лицензия и ограничения upstream

README указывает: "Внутренний проект Automatica. Курсовые материалы не для публичного распространения." Это значит:

- PR с новой фичей — ОК (код инфраструктуры)
- Публикация курсовых материалов в документации — нужно согласование

## Потенциальные проблемы при модификации

### При изменении entrypoint.sh

| Проблема | Описание | Последствие |
| --- | --- | --- |
| Синтаксическая ошибка bash | Один неправильный символ в heredoc | Контейнер не стартует |
| Неверный fallback | Ошибка в `${VAR:-default}` | Неправильный провайдер/ключ |
| Порядок операций | `.claude/.env` создаётся до switch скрипта | Скрипт видит неинициализированный файл |
| Права доступа | Файл создаётся от root, а Claude Code работает от coder | Permission denied |

**Рекомендация:** тестировать entrypoint.sh локально через `docker compose up` после каждого изменения. Добавить `echo` для отладки, убрать перед коммитом.

### При изменении claude-settings.json

| Проблема | Описание | Последствие |
| --- | --- | --- |
| Приоритет env секции | settings.json может перекрыть .claude/.env | Провайдер не переключается |
| Невалидный JSON | Запятая, кавычка — и файл не парсится | Claude Code не запускается |
| Шаблонизация | Если генерировать через envsubst, переменные с `$` в паролях ломаются | Неверные значения |

**Рекомендация:** исследовать приоритеты Claude Code CLI. Если settings.json имеет наивысший приоритет — убрать из него секцию `env` или генерировать динамически.

### При изменении docker-compose.yml

| Проблема | Описание | Последствие |
| --- | --- | --- |
| Маппинг переменных | `ANTHROPIC_AUTH_TOKEN=${GLM_API_KEY:-}` — если переименовать, старые .env не работают | Нет API-ключа в контейнере |
| Healthcheck | Если auth-gateway не стартует из-за ошибки entrypoint | Container unhealthy, restart loop |

**Рекомендация:** поддерживать обратную совместимость с `GLM_*` переменными. Fallback в docker-compose.yml.

### При изменении Dockerfile

| Проблема | Описание | Последствие |
| --- | --- | --- |
| ENV дефолты | Изменение дефолтов ломает контейнеры без .env | Неправильный провайдер |
| Кэш слоёв | Изменение ENV инвалидирует кэш всех последующих слоёв | Долгая пересборка |
| COPY порядок | claude-settings.json копируется до entrypoint.sh | Если генерация переедет — порядок важен |

## На что обратить внимание при тестировании

### Smoke-тесты (обязательные)

- [ ] Контейнер стартует без `.env` файла (дефолты работают)
- [ ] Контейнер стартует с `.env` в старом формате (`GLM_*`)
- [ ] Контейнер стартует с `.env` в новом формате (`PROVIDER=...`)
- [ ] `http://localhost:8080` отдаёт страницу логина
- [ ] После логина открывается code-server
- [ ] В терминале code-server доступна команда `claude`
- [ ] Claude Code использует правильный BASE_URL и модели (проверить через `claude config list` или первый запрос)

### Тесты переключения провайдера

- [ ] `switch-provider.sh zai` — устанавливает Z.AI конфигурацию
- [ ] `switch-provider.sh anthropic` — устанавливает Anthropic конфигурацию
- [ ] После переключения `.claude/.env` содержит правильные значения
- [ ] После переключения Claude Code использует новый провайдер (перезапуск)
- [ ] Переключение без backup ключа — корректное сообщение об ошибке

### Тесты обратной совместимости

- [ ] `.env` с `GLM_API_KEY` работает без изменений
- [ ] `.env` с `GLM_OPUS_MODEL` корректно маппится
- [ ] `switch-api-key.sh` (старый скрипт) продолжает работать или показывает deprecation notice
- [ ] Контейнер без указания PROVIDER использует Z.AI (как раньше)

### Edge cases

- [ ] Пустой API_KEY — контейнер стартует, Claude Code сообщает об ошибке (не крашится)
- [ ] Несуществующий PROVIDER — понятное сообщение об ошибке
- [ ] Спецсимволы в API_KEY (`$`, `!`, `#`) — корректно сохраняются в `.claude/.env`
- [ ] Переключение во время работы Claude Code — что происходит?
- [ ] `docker compose restart` — провайдер сохраняется или сбрасывается?

## Матрица рисков

| Риск | Вероятность | Влияние | Митигация |
| --- | --- | --- | --- |
| claude-settings.json перекрывает .claude/.env | Высокая | Критическое — фича не работает | Исследовать приоритеты до реализации |
| Ошибка в entrypoint.sh ломает старт | Средняя | Критическое — контейнер не работает | Smoke-тест после каждого изменения |
| Старый .env несовместим с новым форматом | Средняя | Высокое — ломает существующие установки | Fallback для GLM_* переменных |
| Спецсимволы в ключах ломают sed/heredoc | Средняя | Среднее — один провайдер не работает | Использовать jq для JSON, экранирование для bash |
| PR не принят в upstream из-за scope | Низкая | Высокое — работа потеряна | Согласовать с мейнтейнером до реализации |
| Документация содержит курсовые материалы | Низкая | Среднее — лицензионный конфликт | Документировать инфраструктуру, не контент курса |
