# План документирования проекта

## Контекст проекта

**Проект:** cc-for-non-coders-dev-container — Docker-контейнер рабочей среды для курса "Claude Code: суперсила для непрограммистов" (Automatica, Season 1).

**Наша роль:** Команда программистов-контрибьюторов. Публичный форк, цель — анализ, документация, PR с улучшениями в upstream.

**Целевые аудитории документации:**

1. **Программисты** (команда контрибьюторов) — подробная техническая документация: архитектура, кодовая база, точки расширения, API, data flow
2. **Непрограммисты** (пользователи курса) — диаграммы, визуализации, простые объяснения на понятном языке

## Ключевые компоненты проекта для документирования

### Инфраструктура

- `Dockerfile` — Ubuntu 22.04, Node.js 22, Python 3, LibreOffice, Playwright, Pandoc, Tesseract OCR, code-server
- `docker-compose.yml` — оркестрация контейнера
- `entrypoint.sh` — скрипт запуска сервисов
- `run.sh` — однокоманд билд/запуск

### Ключевой код

- `auth-gateway.py` (~300 строк) — Python HTTP-прокси с cookie-аутентификацией, главный код проекта
- `login.html` — интерфейс авторизации (тёмная тема)

### Конфигурация

- `.env.example` — шаблон переменных окружения
- `code-server-settings.json` — настройки VS Code
- `claude-settings.json` — разрешения Claude Code CLI

### Учебные материалы

- `course/sessions/` — 5 сессий, 28 демо-проектов
- `skills/` — 19 навыков Claude Code
- `README.md` — описание проекта (226 строк)

### Порты и сетевая архитектура

- `:8080` — auth-gateway.py (внешний, аутентификация)
- `:8081` — code-server (внутренний, VS Code)
- `:9090` — File Browser (внутренний)

## Решения по визуализациям

**Принято:** Mermaid сейчас, Excalidraw потом.

- **Этапы 1-3:** все диаграммы в формате Mermaid (встроено в Markdown, рендерится на GitHub и в VS Code)
- **Позднее:** Excalidraw-визуализации для непрограммистов как отдельная итерация (в BMAD нет встроенной поддержки Excalidraw — потребуется кастомный BMB-workflow или ручная работа)
- **Обоснование:** не блокируем документацию ради формата визуализаций

## Этап 3: Write Document (WD) — Tech Writer

**Загрузить агента:** `/bmad-agent-bmm-tech-writer`
**Агент:** Paige 📚 Technical Writer
**Результат:** дополнительные документы в `docs/`

### Документы для программистов

1. **Architecture Deep Dive** — детальная архитектура с Mermaid-диаграммами
   - Контейнерная архитектура (Docker layers, volumes, networks)
   - Сетевая модель (порты, прокси, аутентификация)
   - Data flow auth-gateway.py
   - Жизненный цикл запроса от браузера до code-server
2. **Developer Guide** — как поднять окружение, структура кода, как вносить изменения
3. **Contributing Guide** — правила PR, стиль кода, тестирование

### Документы для непрограммистов

1. **Визуальный обзор системы** — Mermaid-диаграммы "как это работает" на простом русском языке (минимум узлов, понятные подписи, без технического жаргона)
2. **Карта курса** — визуализация 5 сессий, 28 демо, 19 навыков

### Формат визуализаций

- Все диаграммы — Mermaid в Markdown
- Для непрограммистов — упрощённый стиль: минимум узлов, русские подписи, flowchart без технических деталей
- Excalidraw-версии — отдельная итерация позднее (см. `bmb-integration-plan.md`)

### Инструкция для агента

> Прочитай docs/index.md и _bmad-output/project-context.md. На их основе создай документы для двух аудиторий. Для программистов — детальную архитектуру с Mermaid-диаграммами и developer guide. Для непрограммистов — Mermaid-диаграммы на простом языке без технического жаргона (минимум узлов, русские подписи). Все документы на русском.
