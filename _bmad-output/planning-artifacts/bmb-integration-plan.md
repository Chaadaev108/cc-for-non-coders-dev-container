# План интеграции BMB (Builder)

## Что такое BMB

BMB (BMAD Builder) — мета-модуль для создания кастомных компонентов BMAD:

- **Агенты** — специализированные AI-персоны с экспертизой и опциональной памятью (sidecar)
- **Workflow** — пошаговые процессы с микро-файловой архитектурой (каждый шаг — отдельный файл)
- **Модули** — полные пакеты из агентов + workflow + конфигурации

BMB гарантирует, что всё созданное соответствует стандартам BMAD v6 и может быть валидировано.

## Когда подключается BMB

**Не на этапе документации.** BMB вступает после завершения Этапов 1-3 и реализации фичи провайдеров через Quick Flow.

### Запланированные задачи BMB

#### Задача 1: Workflow для переключения API-провайдеров

**Когда:** после Quick Dev реализации фичи
**Команда:** `/bmad-bmb-create-workflow`
**Агент:** Wendy 🔄 Workflow Building Master

**Цель:** создать BMAD-совместимый workflow, который поможет пользователям (непрограммистам) переключать API-провайдеров через пошаговый процесс.

**Входные артефакты:**

- `_bmad-output/project-context.md`
- `docs/*` (документация проекта)
- реализованный код фичи

**Выходные артефакты:**

- `workflow.md` + `steps-c/step-*.md` — пошаговый workflow
- Шаблоны конфигурации провайдеров
- Документация workflow

#### Задача 2: Workflow для генерации Excalidraw-визуализаций

**Когда:** после Задачи 1 или параллельно
**Команда:** `/bmad-bmb-create-workflow`
**Агент:** Wendy 🔄 Workflow Building Master

**Цель:** создать workflow для генерации `.excalidraw` файлов из документации проекта. Excalidraw использует JSON-формат, который Claude Code способен генерировать.

**Примерные шаги workflow:**

1. Прочитать исходный документ (архитектуру, data flow, и т.д.)
2. Определить тип диаграммы (flowchart, sequence, architecture)
3. Сгенерировать `.excalidraw` JSON с визуально понятными элементами
4. Валидировать JSON-структуру

**Выходные артефакты:**

- Workflow для генерации Excalidraw
- Шаблоны для разных типов диаграмм
- Примеры сгенерированных диаграмм

#### Задача 3 (опционально): Кастомный агент для проекта

**Когда:** после брейншторма по улучшениям
**Команда:** `/bmad-bmb-create-agent`
**Агент:** Bond 🤖 Agent Building Expert

**Потенциальные агенты:**

- Агент для review Docker-конфигов и Dockerfile
- Агент для генерации и обновления учебных материалов курса
- Агент-помощник для студентов курса (отвечает на вопросы простым языком)

**Решение:** принимается после брейншторма (`/bmad-brainstorming`). Сейчас — только фиксация возможности.

## Доступные BMB-команды (справочник)

| Команда | Назначение |
|---------|-----------|
| `/bmad-bmb-create-agent` | Создать нового агента |
| `/bmad-bmb-edit-agent` | Редактировать существующего агента |
| `/bmad-bmb-validate-agent` | Валидировать агента на соответствие стандартам |
| `/bmad-bmb-create-workflow` | Создать новый workflow |
| `/bmad-bmb-edit-workflow` | Редактировать существующий workflow |
| `/bmad-bmb-validate-workflow` | Валидировать workflow |
| `/bmad-bmb-create-module-brief` | Создать бриф для нового модуля |
| `/bmad-bmb-create-module` | Создать полный модуль (агенты + workflow + конфиг) |
| `/bmad-bmb-validate-module` | Валидировать модуль |

## Порядок запуска BMB

| # | Предусловие | Команда | Результат |
|---|-------------|---------|-----------|
| 1 | Фича провайдеров реализована | `/bmad-bmb-create-workflow` | Workflow переключения провайдеров |
| 2 | Документация готова (Этапы 1-3) | `/bmad-bmb-create-workflow` | Workflow генерации Excalidraw |
| 3 | После брейншторма | `/bmad-bmb-create-agent` | Кастомные агенты по итогам обсуждения |
