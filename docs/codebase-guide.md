# Описание кодовой базы

Этот документ подробно описывает каждый исходный файл проекта: что он делает, как устроен, и где искать нужный код.

## Инфраструктурные файлы

### `auth-gateway.py` — HTTP-прокси с авторизацией

**Что это (простыми словами):** Программа-охранник. Когда студент открывает сайт курса, она проверяет пароль и пропускает к рабочему окружению.

**Технические детали:**

- **Язык:** Python 3
- **Размер:** ~316 строк
- **Библиотеки:** только стандартная библиотека (`http.server`, `http.cookies`, `hashlib`, `socket`)
- **Порт:** 8080

**Основные классы и функции:**

| Элемент | Строка (прибл.) | Назначение |
|---------|-----------------|-----------|
| `AuthHandler` | класс | HTTP-обработчик всех запросов |
| `do_GET()` | метод | Обработка GET-запросов (логин, IDE, файлы) |
| `do_POST()` | метод | Обработка POST-запросов (авторизация) |
| `proxy_request()` | метод | Проксирование HTTP к бэкенд-сервисам |
| `handle_websocket()` | метод | WebSocket relay для терминала code-server |
| `check_auth()` | метод | Проверка HMAC-cookie |
| `set_auth_cookie()` | метод | Установка авторизационной cookie |

**Поток авторизации:**

```
1. GET / → нет cookie? → отдать login.html
2. POST /login → пароль верный? → установить HMAC-cookie → redirect /ide/
3. GET /ide/* → cookie валидна? → proxy → code-server:8081
4. GET /files/* → cookie валидна? → proxy → File Browser:9090
```

**WebSocket-проксирование:**
- Используется сырой `socket.socket` для TCP-соединения к code-server
- Двунаправленный relay в отдельном потоке
- Необходимо для работы терминала в VS Code

**Точки расширения для разработчиков:**
- Новые маршруты: добавить `elif` в `do_GET()` / `do_POST()`
- Новый бэкенд-сервис: добавить proxy-правило по аналогии с `/files/`
- JWT вместо HMAC-cookie: заменить `check_auth()` / `set_auth_cookie()`

### `entrypoint.sh` — инициализация контейнера

**Что это (простыми словами):** Скрипт, который запускается при старте контейнера. Подготавливает рабочее окружение студента и запускает все сервисы.

**Технические детали:**

- **Язык:** Bash
- **Размер:** ~81 строка

**Последовательность действий:**

```
1. Проверить, есть ли файлы в /home/coder/course (Docker volume)
2. Если нет → скопировать из pristine-образа (.course-image/)
3. Записать API-ключ и модели в ~/.claude/.env
4. Создать скрипт switch-api-key.sh
5. Добавить welcome-баннер в .bashrc
6. Запустить File Browser (фон, порт 9090)
7. Запустить code-server (фон, порт 8081)
8. Запустить auth-gateway.py (основной процесс)
```

**Точки расширения:**
- Новый сервис: добавить запуск перед `exec` auth-gateway
- Изменение конфигурации: модифицировать блок записи `.env`
- Добавление приветственного сообщения: изменить блок `.bashrc`

### `Dockerfile` — сборка образа

**Что это (простыми словами):** Рецепт для создания контейнера. Описывает, какие программы установить и какие файлы скопировать.

**Структура (по слоям):**

```
1. Базовый образ: Ubuntu 22.04
2. Системные пакеты:
   - LibreOffice (Writer, Calc, Impress)
   - FFmpeg, Pandoc, Poppler-utils, qpdf
   - Tesseract OCR (+ русский язык)
   - tmux, curl, git, jq
3. code-server 4.109.2
4. Node.js 22 (через NodeSource)
5. File Browser (веб-файлменеджер)
6. Claude Code CLI (npm)
7. Глобальные npm-пакеты:
   - docx, pptxgenjs (создание документов)
   - parcel, html-inline (сборка HTML)
8. Python-пакеты (pip):
   - Документы: pypdf, python-pptx, python-docx, openpyxl
   - Данные: pandas, numpy, matplotlib
   - AI: anthropic, mcp
   - Графика: pillow, cairosvg, imageio
   - Парсинг: lxml, pdfplumber, markitdown
   - Браузер: playwright
9. Пользователь: coder (без root)
10. Playwright + Chromium
11. Расширения VS Code: Live Server, PDF/Excel viewer, Draw.io
12. Копирование курса и навыков
13. Кеширование: Chart.js, @anthropic-ai/model-context-protocol
14. Переменные окружения
```

**Точки расширения:**
- Новые системные пакеты: добавить в блок `apt-get install`
- Новые npm-пакеты: добавить в `npm install -g`
- Новые Python-пакеты: добавить в `pip3 install`
- Новые VS Code-расширения: добавить `code-server --install-extension`

### `docker-compose.yml` — оркестрация

**Что это (простыми словами):** Файл, который описывает, как запустить контейнер — какие порты открыть, какие переменные передать, сколько ресурсов выделить.

**Ключевые параметры:**

```yaml
services:
  student:
    container_name: claude-course-test
    ports: 8080:8080
    volumes: student-data:/home/coder/course
    deploy:
      resources:
        limits:
          cpus: '2'
          memory: 4G
```

**Переменные из `.env`:**
- `PASSWORD` — пароль для входа
- `GLM_API_KEY` — основной API-ключ
- `GLM_*_MODEL` — переопределения моделей
- `API_TIMEOUT_MS` — таймаут
- `GLM_API_KEY_BACKUP` — резервный ключ

### `run.sh` — быстрый запуск

**Что это (простыми словами):** Скрипт для быстрого запуска всего проекта одной командой.

**Действия:**
1. Подсчитывает файлы в `course/` и `skills/`
2. Выполняет `docker compose build`
3. Выполняет `docker compose up -d`
4. Выводит URL и пароль для входа

---

### `login.html` — страница входа

**Что это (простыми словами):** Красивая тёмная страница с формой для ввода пароля.

**Технические детали:**
- Встроенные стили (CSS в `<style>`)
- Фон: #0f0f1a, карточка: #1a1a2e
- Фиолетовая кнопка (#a855f7)
- Плейсхолдер `<!--ERROR-->` заменяется серверной стороной (auth-gateway.py)
- Адаптивный дизайн (flexbox)

---

### `claude-settings.json` — настройки безопасности

**Что это (простыми словами):** Файл, который определяет, что Claude Code может и не может делать в контейнере.

**Структура:**
- `permissions.allow` — белый список разрешённых инструментов
- `permissions.deny` — чёрный список запрещённых команд
- `env` — переменные окружения (API URL, модели, таймаут)

## Учебные файлы

### `course/CLAUDE.md` — контекст курса для AI

**Назначение:** Инструкции для Claude Code при работе внутри каталога курса. Определяет правила поведения, стиль общения, запреты.

### `course/course-outline.md` — план курса

**Назначение:** Детальный план всех 5 сессий (~400 строк), включая тайминги, темы, описания демо. Вторая часть — маркетинговый лендинг курса.

### Демо-проекты (`course/sessions/*/demo/`)

**Каждый демо содержит:**
- `README.md` — инструкции для преподавателя (что показать, какие промпты использовать, ожидаемый результат, troubleshooting)
- Данные: CSV, MD, JSON (в зависимости от задачи)
- Опционально: `CLAUDE.md` — контекст для AI при работе над демо

## Навыки (`skills/`)

Каждый навык — это каталог с файлом `SKILL.md`, содержащим:
- Мета-данные (имя, описание, автор, триггеры)
- Полные инструкции для Claude Code (промпты, шаги, правила)
- Ссылки на вспомогательные скрипты и шаблоны

Подробнее о навыках: [Структура курса и навыков](./course-and-skills.md).
