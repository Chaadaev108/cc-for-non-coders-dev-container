### Как устроен навык

Каждый навык — это каталог в `skills/` со следующей структурой:

```
skill-name/
├── SKILL.md       # Обязательно: инструкции для Claude Code
├── LICENSE.txt    # Обязательно: лицензия
├── references/    # Опционально: справочные материалы (.md)
├── scripts/       # Опционально: вспомогательные скрипты (.py, .js)
├── templates/     # Опционально: шаблоны
└── themes/        # Опционально: темы/конфигурации
```

**SKILL.md** содержит:
- Мета-данные (имя, описание, версия, триггеры)
- Пошаговые инструкции для Claude Code
- Правила и ограничения
- Примеры использования

### Установка навыков в контейнер

Навыки копируются в два места при сборке Docker-образа:
1. `~/.claude/skills/` — для доступа из Claude Code CLI
2. `course/.claude/skills/` — для доступа из рабочего каталога курса

Студенты могут также устанавливать навыки с GitHub (демо `github-skill-install`).

## Структура демо-проекта

Каждый демо-проект следует единому шаблону:

```
demo-name/
├── README.md      # Инструкции преподавателя
│                  # - Что показать
│                  # - Какие промпты использовать
│                  # - Ожидаемый результат
│                  # - Troubleshooting
├── CLAUDE.md      # Опционально: контекст для AI
└── data-files     # CSV, MD, JSON — входные данные
```

**README.md преподавателя** содержит:
- Цель демо
- Пошаговые инструкции (промпты, действия)
- Ожидаемый результат (с примерами)
- Возможные проблемы и их решения
- Варианты расширения задачи
