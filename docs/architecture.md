# Архитектура системы

## Обзор

Проект представляет собой Docker-контейнер с тремя внутренними сервисами, объединёнными через reverse-proxy шлюз с аутентификацией. Контейнер предоставляет полностью изолированную учебную среду для работы с Claude Code.

## Компоненты системы

### 1. Auth Gateway (`auth-gateway.py`)

**Назначение:** Единая точка входа с аутентификацией и проксированием.

**Технология:** Python 3, `http.server`, `http.cookies`

**Ключевые функции:**
- HMAC-SHA256 аутентификация через cookie
- Маршрутизация запросов к code-server и File Browser
- Полная поддержка WebSocket (для терминала code-server)
- Отдача login.html при отсутствии авторизации
- Healthcheck endpoint (`/healthz`)

**Маршруты:**

| Путь | Метод | Назначение | Бэкенд |
|------|-------|-----------|--------|
| `/` | GET | Логин / редирект в IDE | — |
| `/login` | POST | Авторизация | — |
| `/logout` | GET | Выход | — |
| `/healthz` | GET | Health check | — |
| `/ide/*` | GET/POST/WS | Проксирование в code-server | `localhost:8081` |
| `/files/*` | GET/POST | Проксирование в File Browser | `localhost:9090` |

**Безопасность:**
- Пароль задаётся через переменную окружения `PASSWORD`
- Cookie подписывается HMAC-SHA256 (секрет = пароль)
- Если `PASSWORD` пуст — авторизация пропускается (dev-режим)

### 2. Code-server

**Назначение:** Веб-версия VS Code, доступная через браузер.

**Порт:** 8081 (внутренний, проксируется через gateway)

**Настройки** (`code-server-settings.json`):
- Шрифт 14, тёмная тема (Dark+)
- Авто-сохранение через 1 секунду
- Минимэп отключён
- Терминал: bash, шрифт 14

**Расширения VS Code:**
- Live Server (для просмотра HTML)
- PDF Viewer
- Excel Viewer
- Draw.io

### 3. File Browser

**Назначение:** Веб-файловый менеджер для загрузки/скачивания файлов.

**Порт:** 9090 (внутренний, проксируется через `/files/`)

**Режим:** без аутентификации (noauth), авторизация на уровне gateway.

### 4. Claude Code CLI

**Назначение:** AI-ассистент для работы с кодом и документами.

**Конфигурация** (`claude-settings.json`):
- API-эндпоинт: `https://api.z.ai/api/anthropic`
- Маппинг моделей: claude-sonnet-4-5-20250514 → GLM-5, claude-haiku-4-5-20251001 → GLM-4.5-Air
- Таймаут: 3 000 000 мс
- Разрешённые команды: ls, find, cat, head, tail, python3, node, npm, git и др.
- Запрещённые команды: `rm -rf /`, `sudo:*`, `chmod 777:*`, `curl:*|wget:*`

## Сетевая архитектура

```
Внешний мир → :8080 (auth-gateway)
                ├── HTTP → code-server :8081 (с strip prefix /ide/)
                ├── WebSocket → code-server :8081 (полный relay)
                └── HTTP → File Browser :9090 (с сохранением prefix /files/)
```

Все сервисы работают на `localhost` внутри одного контейнера. Наружу экспонируется только порт 8080.

## Жизненный цикл контейнера

### Инициализация (`entrypoint.sh`)

1. Проверяет наличие курсовых материалов в volume
2. Если volume пуст — копирует из pristine-образа (`/home/coder/.course-image/`)
3. Записывает API-ключ и конфигурацию моделей в `~/.claude/.env`
4. Создаёт утилиту `switch-api-key.sh` для переключения API-ключей
5. Добавляет welcome-баннер в `.bashrc`
6. Запускает File Browser (фон)
7. Запускает code-server (фон)
8. Запускает auth-gateway.py (основной процесс контейнера)

### Переменные окружения

| Переменная | Обязательная | Описание |
|-----------|-------------|----------|
| `PASSWORD` | Да | Пароль для входа |
| `GLM_API_KEY` | Да | API-ключ Z.AI |
| `GLM_OPUS_MODEL` | Нет | Переопределение модели Opus |
| `GLM_SONNET_MODEL` | Нет | Переопределение модели Sonnet |
| `GLM_HAIKU_MODEL` | Нет | Переопределение модели Haiku |
| `API_TIMEOUT_MS` | Нет | Таймаут API (по умолчанию 3000000) |
| `GLM_API_KEY_BACKUP` | Нет | Резервный API-ключ |

### Ресурсные ограничения (docker-compose.yml)

- CPU: 2 ядра
- RAM: 4 GB
- Volume: `student-data` → `/home/coder/course` (персистентность данных студента)

## Стратегия данных

- **Pristine-образ** (`/home/coder/.course-image/`): нетронутая копия курсовых материалов, вшита в Docker image
- **Рабочий volume** (`/home/coder/course`): монтируется как named volume, данные студента сохраняются между перезапусками
- **Навыки** (`~/.claude/skills/`): копируются в image при сборке, также доступны в `course/.claude/skills/`
- **Сброс**: чистый старт через пересоздание volume

## Безопасность

### Сетевая изоляция
- Все внутренние сервисы доступны только через auth-gateway
- WebSocket relay полностью проксируется через авторизованное соединение

### Ограничения Claude Code
- Белый список разрешённых команд в `claude-settings.json`
- Блокировка: `rm -rf /`, `sudo`, `chmod 777`, `curl`/`wget` (pipe)
- Пользователь `coder` без root-прав (sudo с NOPASSWD для установки пакетов при необходимости)

### Аутентификация
- HMAC-SHA256 cookie, срок жизни не ограничен (до logout/перезапуска)
- Пароль задаётся в `.env`, не хардкодится
