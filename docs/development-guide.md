# Руководство по разработке

## Предварительные требования

| Инструмент | Минимальная версия | Назначение |
|-----------|-------------------|-----------|
| Docker | 20.x | Сборка и запуск контейнера |
| Docker Compose | 2.x | Оркестрация сервисов |
| Git | 2.x | Управление версиями |
| Текстовый редактор | любой | Редактирование конфигурации |

## Быстрый старт

### 1. Клонирование репозитория

```bash
git clone <url-вашего-форка>
cd cc-for-non-coders-dev-container
```

### 2. Настройка окружения

```bash
cp .env.example .env
```

Отредактируйте `.env`:

```bash
PASSWORD=student-2026         # Пароль для входа
GLM_API_KEY=your-api-key      # API-ключ Z.AI
```

### 3. Сборка и запуск

```bash
# Способ 1: через run.sh
chmod +x run.sh
./run.sh

# Способ 2: вручную
docker compose build
docker compose up -d
```

### 4. Проверка

Откройте в браузере: `http://localhost:8080`

Введите пароль из `.env` → попадёте в IDE.

### Healthcheck

```bash
curl http://localhost:8080/healthz
# Ожидаемый ответ: 200 OK
```

## Управление контейнером

### Основные команды

```bash
# Запуск
docker compose up -d

# Остановка
docker compose down

# Пересборка (после изменений в Dockerfile)
docker compose build --no-cache && docker compose up -d

# Логи
docker compose logs -f

# Вход в контейнер
docker exec -it claude-course-test bash

# Вход как root
docker exec -it -u root claude-course-test bash
```

### Сброс данных студента

```bash
# Удалить volume с данными
docker compose down -v

# При следующем запуске — данные будут скопированы из pristine-образа
docker compose up -d
```

### Переключение API-ключа

Внутри контейнера:

```bash
switch-api-key.sh  # Переключает между основным и резервным ключом
```

## Разработка

### Структура проекта для контрибьюторов

```
Инфраструктура (редко меняется):
├── Dockerfile              ← Изменения требуют пересборки образа
├── entrypoint.sh           ← Изменения вступают при перезапуске
├── auth-gateway.py         ← Изменения вступают при перезапуске
├── docker-compose.yml      ← Изменения при `docker compose up -d`

Контент (меняется часто):
├── course/                 ← Материалы курса
│   ├── sessions/*/demo/    ← Демо-проекты
│   └── course-outline.md   ← План курса

Навыки (меняются умеренно):
└── skills/                 ← Навыки Claude Code
```

### Добавление нового демо

1. Создайте каталог в соответствующей сессии:
   ```
   course/sessions/XX-topic/demo/new-demo/
   ```

2. Создайте `README.md` с инструкциями преподавателя:
   ```markdown
   # Название демо

   ## Цель
   Что студент научится делать.

   ## Подготовка
   Необходимые данные и настройки.

   ## Демонстрация
   ### Шаг 1
   Промпт: ...
   Ожидаемый результат: ...

   ## Troubleshooting
   - Проблема: ...
     Решение: ...
   ```

3. Добавьте данные (CSV, MD, JSON).

4. Опционально: создайте `CLAUDE.md` с контекстом.

### Добавление нового навыка (skill)

1. Создайте каталог в `skills/`:
   ```
   skills/my-skill/
   ```

2. Создайте `SKILL.md`:
   ```markdown
   ---
   name: my-skill
   description: Что навык делает
   version: 1.0.0
   ---

   # Инструкции для Claude Code
   ...
   ```

3. Добавьте `LICENSE.txt`.

4. Опционально: добавьте `references/`, `scripts/`, `templates/`.

5. Обновите README.md — добавьте навык в список.

### Добавление системного пакета в контейнер

1. Откройте `Dockerfile`
2. Найдите блок `apt-get install`
3. Добавьте название пакета
4. Пересоберите: `docker compose build --no-cache`

### Добавление Python-пакета

1. Откройте `Dockerfile`
2. Найдите блок `pip3 install`
3. Добавьте название пакета
4. Пересоберите образ

### Добавление npm-пакета

1. Откройте `Dockerfile`
2. Найдите блок `npm install -g`
3. Добавьте название пакета
4. Пересоберите образ

## Тестирование

### Ручное тестирование

Проект не содержит автоматизированных тестов. Тестирование проводится вручную:

1. **Сборка образа**: `docker compose build` — должна пройти без ошибок
2. **Запуск контейнера**: `docker compose up -d` — healthcheck должен вернуть 200
3. **Авторизация**: Вход через `/login` с паролем из `.env`
4. **IDE**: Проверить открытие code-server через `/ide/`
5. **Файлы**: Проверить доступ к File Browser через `/files/`
6. **Claude Code**: Запустить `claude` в терминале — должен работать с API
7. **Навыки**: Вызвать навык (например `/pdf`) — должен работать

### Healthcheck

Docker Compose настроен с healthcheck:

```yaml
healthcheck:
  test: ["CMD", "curl", "-f", "http://localhost:8080/healthz"]
```

## Развёртывание

### Production-режим

Для запуска на сервере:

1. Установите Docker и Docker Compose
2. Склонируйте репозиторий
3. Настройте `.env` (установите надёжный пароль)
4. Запустите `./run.sh`

### Важные замечания

- Контейнер рассчитан на **одного студента** (один экземпляр = один пользователь)
- Для группы студентов запускайте несколько контейнеров с разными портами
- Volume `student-data` сохраняет работу студента между перезапусками
- Ресурсные лимиты: 2 CPU, 4 GB RAM (можно изменить в `docker-compose.yml`)
